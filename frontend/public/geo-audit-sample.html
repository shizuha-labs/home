1.2 Indexing & Crawlability

🔴 SPA Shell Returns Generic Content to Crawlers

Public fetches of shizuha.com return a minimal SPA shell ("Shizuha") with no meaningful content, title tags, or meta descriptions. AI engines and search crawlers see an empty page, severely limiting indexable surface area.

✅ Fix: Implement server-side rendering (SSR) or static pre-rendering for key pages. Ensure <title>, <meta description>, and body content are present in initial HTML.

🟡 robots.txt and sitemap.xml Return SPA Shell

Both /robots.txt and /sitemap.xml return the generic SPA shell instead of their expected content. Crawlers cannot discover the site's page structure or crawl directives.

✅ Fix: Serve actual robots.txt (with sitemap reference) and sitemap.xml from the edge/CDN, bypassing the SPA router.

🔵 No Open Graph or Twitter Card Meta Tags

Missing og:title, og:description, og:image, and twitter:card tags. When Shizuha links are shared on social platforms or cited by AI engines that scrape link previews, they show as blank or generic.

✅ Fix: Add OG/Twitter meta tags to all public-facing pages with descriptive titles, summaries, and branded images.

2.2 Content Depth Gaps

🔴 No Dedicated "About" or "Products" Pages

The site lacks dedicated pages describing what Shizuha builds, who the team is, or what products/services are offered. Without these pages, AI engines have no authoritative source to cite for basic company questions.

✅ Fix: Create /about, /products, and /team pages with rich, entity-dense content. Include schema.org Organization, Person, and Product markup.

🟡 No Blog or Thought-Leadership Content

AI engines heavily cite fresh, authoritative content. Shizuha has no visible blog, case studies, or technical articles on the primary domain that could establish topical authority.

✅ Fix: Launch a /blog section with regular technical and product content. Each post should include Article schema markup and internal links to product pages.

🟡 No FAQ or Help Center

FAQ pages are the highest-converting content type for AI answer inclusion. Shizuha has no FAQ surface, missing the easiest path to appearing in AI-generated answers.

✅ Fix: Create a /faq page with common questions about Shizuha's products, pricing, and capabilities. Use FAQPage schema markup.

🔵 GitHub Repos Are the Primary Source of Truth

Most discoverable information about Shizuha comes from GitHub repositories (shizuha-labs), not the company website. This means AI engines associate Shizuha with code, not with its products or services.

✅ Fix: Cross-link GitHub repos from the website. Add "Code" and "Open Source" sections that reference key repos, creating a bidirectional entity graph.

3. Citation-Readiness Fixes

For AI engines to cite Shizuha confidently, the site needs to provide clear, structured, and authoritative information that matches common user queries.

3.1 Structured Data Implementation

🔴 Zero JSON-LD Structured Data

No JSON-LD or microdata markup exists on any page. This is the single biggest gap — structured data is the primary mechanism by which AI engines extract entity information for citation.

✅ Fix: Implement Organization JSON-LD on all pages: 
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "Shizuha Global Pvt. Ltd.",
  "url": "https://shizuha.com",
  "logo": "https://shizuha.com/logo.png",
  "sameAs": [
    "https://github.com/shizuha-labs",
    "https://x.com/shizuha"
  ],
  "founder": {
    "@type": "Person",
    "name": "Hritik Soni"
  },
  "description": "AI agent platform and developer tools for autonomous workflows",
  "knowsAbout": ["Artificial Intelligence", "Developer Tools", "Agent Platforms"]
}

3.2 Content Authority Signals

🟡 No Authoritative "About" Content for Entity Resolution

AI engines need a clear, self-described "who we are" page to resolve entity ambiguity. Without it, Shizuha may be confused with other entities sharing the name.

✅ Fix: Create an /about page with: company description, founding story, team bios, mission statement, and clear differentiation from competitors. Include Person schema for key team members.

🟡 No Product/Service Descriptions with Structured Data

AI engines can't answer "What does Shizuha offer?" because there are no product pages with structured descriptions, pricing, or feature lists.

✅ Fix: Create product pages (/products/hive, /products/pulse, etc.) with SoftwareApplication schema, feature lists, and clear value propositions.

🔵 Missing Backlink Profile from Authority Sites

AI engines weigh backlinks from authoritative sources. Shizuha's backlink profile is limited to GitHub and developer forums, missing industry publications, directories, and news mentions.

✅ Fix: Submit to relevant directories (Crunchbase, Product Hunt, G2). Pursue press coverage and guest posts on AI/developer blogs to build authority backlinks.
